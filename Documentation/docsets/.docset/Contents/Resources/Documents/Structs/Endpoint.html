Endpoint With Request & Response
  • WithRequestAndResponseType

    Endpoint that takes its generic parameter RequestDataType, encodes it to Data and sets it as http request body.

    In order to user this kind of endpoint, you have to:

    • Subclass it and set specific RequestDataType and ResponseDataType
    • Override whichever computed properties you like from the class to configure your network target
    • Call the initialiser of the Subclass and pass it request data of RequestDataType and completion block with ResponseDataType and Endpont.Error

    By default Codable JSONs are supported, but if you want you can provide your custom encoding by overriding the encodeRequest() function.

  • WithResponseTypeOnly

    Endpoint that does not set http request body.

    In order to user this kind of endpoint, you have to:

    • Subclass it and set specific ResponseDataType
    • Override whichever computed properties you like from the class to configure your network target
    • Call the initialiser of the Subclass and pass it completion block with ResponseDataType and Endpont.Error

    By default Codable JSONs are supported, but if you want you can provide your custom encoding by overriding the decodeResponse() function.
